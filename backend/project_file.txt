1. PROJECT SUMMARY

GemAI Studio is a web application for jewellery retailers, photographers, and e-commerce teams to upload jewellery images, enhance them, clean backgrounds, generate studio-quality output, and perform virtual try-on on model photos.

The product is optimized for both desktop and mobile web use. It follows a single-AI-provider strategy to reduce complexity, cost, and integration risk. The recommended AI model is Google Gemini 3 Pro Image (Nano Banana Pro) used as the only external AI API for image understanding and image editing. All other functionality is handled by the application itself using Node.js services, image processing libraries, and database-backed workflows.

The platform is designed for Hostinger deployment.

2. PRODUCT GOALS

7. AI STRATEGY

Single AI API Approach:
Use only one external AI API for all image intelligence and editing: Google Gemini 3 Pro Image (Nano Banana Pro).

Use cases for the AI model:
- Understand jewellery type and structure
- Analyze image quality issues
- Support prompt-based image editing
- Support guided image-to-image enhancement
- Assist with safe transformation instructions
- Help with virtual try-on guidance and composition suggestions

Non-AI image tasks remain local:
- Cropping
- Resizing
- Compression
- Thumbnail generation
- Mask handling
- Alpha compositing
- Shadow rendering
- Watermark stamping
- EXIF stripping
- File validation

Important rule:
The application must never use text-to-image generation for product transformation. All edits should be image-preserving and constrained to the uploaded source image.

9. SYSTEM ARCHITECTURE

Client Browser
-> Next.js Frontend
-> Node.js API
-> PostgreSQL
-> Redis Queue
-> Local Image Worker
-> Gemini 3 Pro Image API
-> Storage
-> Download / Share

Processing pattern:
1. User uploads file
2. Frontend validates file and sends to backend
3. Backend stores original file and metadata
4. Backend creates a job record
5. Queue worker processes the job
6. Worker calls Gemini API when AI analysis or editing is needed
7. Worker performs local image operations
8. Result is saved to storage
9. Backend updates job status
10. Frontend receives real-time updates and shows the output

19. RISKS AND MITIGATIONS

- Risk: AI changes jewellery shape
  Mitigation: constrained image editing only, no text-to-image, strong validation, human review option

- Risk: Try-on alignment looks fake
  Mitigation: manual repositioning, blend controls, clear confidence warnings

- Risk: Upload errors on mobile
  Mitigation: lightweight responsive UI, resumable upload support if needed

- Risk: Hostinger resource limits
  Mitigation: separate worker process, optimize image size, cache aggressively, scale to VPS plan as usage grows
