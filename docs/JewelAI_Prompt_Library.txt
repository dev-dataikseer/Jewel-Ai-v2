How to Use This Document

Every prompt in this document is production-ready. Copy the prompt
exactly as written, then replace the [BRACKETED] variables with the
values the user selects in the UI. Each prompt section includes:

1.  The prompt text — formatted in a dark code block exactly as it
    should be sent to the AI API.

2.  A variable table — listing every [BRACKET] in the prompt, what it
    means, and example values.

3.  A developer note — implementation tips, defaults, and edge cases to
    handle.

For image generation features (1, 2, 3, 4, 5, 6, 7, 10, 11): send the
prompt to an image AI API (Stable Diffusion, DALL-E, Midjourney API, or
similar) alongside the user's uploaded reference image.

For text/data features (8, 9): send the prompt to the Claude API or
similar LLM and display the text response in the UI.

FEATURE 1 — PUT JEWELLERY ON A MODEL

These prompts are used when the user uploads a jewellery image and
selects a jewellery type. The system places the piece on a realistic
human model. The prompt is auto-built from the user's selections — the
developer plugs the selected values into [BRACKETS].

1A — Necklace on Model

Prompt:

A professional jewellery model photograph. A [SKIN_TONE] female model
wearing a [METAL_TYPE] [NECKLACE_TYPE] necklace set with [GEMSTONE]
gemstones, draped elegantly around her neck. The model is posed in a
[POSE] position against a clean [BACKGROUND] background. Studio lighting
with soft shadows. Ultra-sharp focus on the necklace. Shot on a high-end
DSLR camera. Editorial fashion photography style. The necklace from the
reference image is placed accurately on the model's neckline.
Photorealistic, 8K, luxury jewellery advertisement.

Variables:

  ----------------- ---------------------------------- ------------------------------------------------------
  Variable          What it represents                 Example values
  [SKIN_TONE]       Model skin tone selected by user   fair, wheatish, dusky, dark
  [METAL_TYPE]      Metal of the jewellery             gold, rose gold, silver, platinum
  [NECKLACE_TYPE]   Type of necklace                   choker, layered, long pendant, collar, chain
  [GEMSTONE]        Primary stone in piece             diamond, ruby, emerald, pearl, kundan
  [POSE]            Model pose style                   straight-on, slight three-quarter turn, looking down
  [BACKGROUND]      Studio background preference       pure white, soft grey, cream, gradient beige
  ----------------- ---------------------------------- ------------------------------------------------------

  📝 Developer Note: If user does not specify skin tone, default to
  wheatish. If background not selected, default to pure white.

FEATURE 2 — CUSTOMER TRY ON

The user uploads a customer's photo AND a jewellery image. The AI places
the jewellery accurately on the customer. The prompt tells the AI
exactly where to place the piece and how to blend it naturally. The
jewellery type drives the placement instruction.

2A — Necklace Try On

Prompt:

Take the jewellery piece from the reference jewellery image and place it
realistically and accurately on the person in the customer photo. The
necklace should sit naturally around the person's neckline, following
the curve of their neck. Match the lighting, shadow direction, and skin
reflection from the customer photo. The metal should reflect light
consistent with the photo's ambient lighting. Preserve the exact design,
metalwork, and gemstone colours of the reference jewellery. Do not alter
the customer's face, skin, clothing, or any other part of the image. The
result must look like a real photograph — not a digital composite.
Photorealistic, high resolution, natural blending.

  📝 Developer Note: This prompt works as-is. The two uploaded images
  (customer photo + jewellery image) are passed as reference images
  alongside this prompt.

2B — Ring Try On

Prompt:

Take the ring from the reference jewellery image and place it
realistically on the ring finger of the right hand of the person in the
customer photo. The ring must sit naturally on the finger with accurate
sizing relative to the finger width. Match the lighting, shadow, and
skin tone from the customer photo. The metal and gemstone should reflect
light consistent with the photo environment. Preserve the exact design,
stone colour, and metalwork of the reference ring. Do not alter any
other part of the customer image. The result must look like a real
photograph taken while wearing the ring. Photorealistic, high
resolution, natural blending.

  📝 Developer Note: If the customer's hand is not clearly visible in the
  photo, the system should prompt the user to upload a clear hand photo
  instead of processing a bad result.

2C — Earrings Try On

Prompt:

Take the earrings from the reference jewellery image and place them
realistically on both earlobes of the person in the customer photo. The
earrings must hang naturally at the correct angle based on the person's
head tilt and ear position. Match the lighting, shadow direction, and
skin tone from the customer photo. The metal and stones should reflect
light consistent with the photo's ambient lighting. Preserve the exact
design, colour, and proportions of the reference earrings. Do not alter
the customer's face, hair, or any other part of the image. The result
must look like a real photograph. Photorealistic, high resolution,
natural blending.

  📝 Developer Note: If the customer's ears are obscured by hair, add to
  the prompt: 'If an ear is not visible, place only the visible earring
  and leave the hidden ear untouched.'

2D — Bangle / Bracelet Try On

Prompt:

Take the bangle or bracelet from the reference jewellery image and place
it realistically on the wrist of the person in the customer photo. The
bangle must sit naturally around the wrist with correct sizing and a
slight gap from the skin as a real bangle would appear. Match the
lighting, shadow, and skin tone from the customer photo. Preserve the
exact design, metalwork, and gemstone colours. Do not alter any other
part of the customer image. The result must look like a real photograph
taken while wearing the jewellery. Photorealistic, high resolution,
natural blending.

  📝 Developer Note: If user selects a bangle set, duplicate the bangle
  3–5 times stacked on the wrist.

FEATURE 3 — GEMSTONE COLOUR CHANGE

The user uploads a jewellery image, selects the current gemstone colour,
and selects the target gemstone colour. The AI replaces only the stone
colour while keeping the metal, setting, and all other elements
completely unchanged.

3A — Master Gemstone Swap Prompt

Prompt:

In the reference jewellery image, identify all [SOURCE_STONE] coloured
gemstones. Replace their colour and visual appearance with
[TARGET_STONE] gemstones of the same cut, size, and setting position.
The replacement stones must have the correct colour, transparency,
brilliance, and light refraction properties of a real [TARGET_STONE]
gemstone. The metal setting, metalwork design, prongs, and all non-stone
elements must remain completely unchanged. Do not alter the background,
lighting, or composition. The final image must be photorealistic and
indistinguishable from a real photograph of the jewellery set with
[TARGET_STONE] stones. 8K resolution, ultra-sharp focus.

Variables:

  ---------------- ----------------------------------- --------------------------------------------------------------------------------
  Variable         What it represents                  Example values
  [SOURCE_STONE]   The current gemstone in the image   Ruby (red), Emerald (green), Blue Sapphire (blue), Pearl (white), Onyx (black)
  [TARGET_STONE]   The desired replacement gemstone    Yellow Sapphire, Pink Tourmaline, White Diamond, Amethyst, Coral, Tsavorite
  ---------------- ----------------------------------- --------------------------------------------------------------------------------

  📝 Developer Note: Build a dropdown UI mapping stone names to colour
  descriptions. Pass both the name and colour to get accurate results.

FEATURE 4 — CATALOGUE IMAGE GENERATOR

The user uploads a raw jewellery photo taken on a phone. The AI removes
the background, isolates the jewellery, and places it on a clean
professional background. The prompt changes based on the background
style selected.

4A — Pure White Background

Prompt:

Take the jewellery piece from the reference image. Remove the existing
background completely. Place the jewellery item centred on a pure white
(#FFFFFF) seamless studio background. The jewellery should cast a very
soft, natural shadow directly beneath it. Adjust the lighting to be
soft, even, and shadowless on the jewellery itself, highlighting all
metalwork and gemstone facets. Correct any colour cast from the original
photo. The final result must look like a professional product photograph
taken in a high-end studio. Sharp focus throughout, 8K resolution, clean
edges, no reflections on the background.

  📝 Developer Note: Pure white is the default. Use this prompt when no
  background is selected.

4B — Cream / Ivory Background

Prompt:

Take the jewellery piece from the reference image. Remove the existing
background completely. Place the jewellery item centred on a warm cream
(#F5F0E8) seamless studio background. The jewellery should cast a soft,
warm shadow beneath it. Use warm, soft studio lighting that enhances the
warmth of gold tones and the richness of coloured stones. The final
result must look like a luxury jewellery catalogue photograph. Sharp
focus, 8K resolution, clean edges, photorealistic.

  📝 Developer Note: Cream background works best for yellow gold and
  pearl jewellery.

4C — Black / Dark Background

Prompt:

Take the jewellery piece from the reference image. Remove the existing
background completely. Place the jewellery item centred on a deep black
(#0A0A0A) seamless background. Use dramatic side lighting with a subtle
fill light to highlight the brilliance of the gemstones and the lustre
of the metal against the dark background. The jewellery should appear to
float slightly with a faint glow around it. Ultra-sharp focus on all
facets and metalwork details, 8K resolution, luxury editorial style.

  📝 Developer Note: Black background is ideal for diamond and high-shine
  jewellery. Not recommended for dark-stone pieces.

4D — Marble Texture Background

Prompt:

Take the jewellery piece from the reference image. Remove the existing
background completely. Place the jewellery item centred on a clean white
Carrara marble surface with subtle grey veining. The jewellery should
rest directly on the marble surface and cast a soft, natural shadow. Use
soft overhead studio lighting. The marble pattern should be subtle and
not compete with the jewellery. Sharp focus on the jewellery, gentle
blur on the marble edges, 8K resolution, luxury product photography.

  📝 Developer Note: Marble background suits bridal and high-end wedding
  jewellery collections.

4E — Gradient Background

Prompt:

Take the jewellery piece from the reference image. Remove the existing
background completely. Place the jewellery item centred on a smooth
gradient background transitioning from [GRADIENT_START] at the top to
[GRADIENT_END] at the bottom. Soft, even studio lighting. The jewellery
should cast a minimal soft shadow. Clean, sharp focus on all details, 8K
resolution, modern product photography.

Variables:

  ------------------ ------------------------------- --------------------------------------------------------
  Variable           What it represents              Example values
  [GRADIENT_START]   Top colour of the gradient      soft blush pink, light champagne, pale grey, soft gold
  [GRADIENT_END]     Bottom colour of the gradient   deep rose, warm beige, mid grey, deep amber
  ------------------ ------------------------------- --------------------------------------------------------

  📝 Developer Note: Popular gradient combos: blush-to-rose (for
  rosegold), champagne-to-amber (for yellow gold), grey-to-charcoal (for
  platinum/silver).

FEATURE 5 — BULK CATALOGUE GENERATION

Bulk generation uses the same prompts as Feature 4. The only difference
is that the same selected background prompt is applied to every image in
the batch. No separate prompts are needed — the developer runs Feature 4
prompts in parallel for each uploaded image.

5A — Batch Instruction (System-Level, Not User-Facing)

Prompt:

Process each jewellery image in the batch independently using the
following instruction:

Take the jewellery piece from the image. Remove the existing background
completely. Place the jewellery item centred on a [SELECTED_BACKGROUND]
background. Apply consistent soft studio lighting across all images in
the batch. Each piece should cast a natural soft shadow. Maintain
consistent image dimensions, resolution, and white balance across all
outputs. Sharp focus on every piece, 8K resolution, professional product
photography.

Apply this instruction to all [TOTAL_IMAGES] images. Do not mix elements
between images. Each output represents only the jewellery from its
corresponding input image.

Variables:

  ----------------------- ------------------------------------------------------------ --------------------------------------------
  Variable                What it represents                                           Example values
  [SELECTED_BACKGROUND]   Background style chosen by user, same as Feature 4 options   pure white, cream, black, marble, gradient
  [TOTAL_IMAGES]          Number of images in the batch                                1 to 50
  ----------------------- ------------------------------------------------------------ --------------------------------------------

  📝 Developer Note: Pass each image individually to the AI API in
  parallel requests, all using the same prompt. Combine results
  server-side before returning to the user.

FEATURE 8 — DIAMOND RATE FETCH

This feature is not an AI image prompt — it uses a pricing API or
database. However, a natural language prompt is needed to display the
result clearly and to handle edge cases. The following prompts drive the
conversational display layer.

8A — Rate Display Prompt (Result Formatting)

Prompt:

Based on the following diamond specifications, provide pricing guidance
in a clear, professional format:

Shape: [SHAPE]

Colour Grade: [COLOR]

Clarity Grade: [CLARITY]

Carat Weight: [CARAT]

Discount Applied: [DISCOUNT]%

Return the estimated price range in INR and USD. Explain in 2 sentences
what factors most influence this specific stone's value. Add a
disclaimer that prices are indicative and vary by supplier and market
conditions. Keep the tone professional and helpful — as if a senior
diamond trader is giving guidance.

Variables:

  ------------ --------------------------- --------------------------------------------------------------------------------
  Variable     What it represents          Example values
  [SHAPE]      Diamond shape               Round Brilliant, Princess, Oval, Pear, Cushion, Emerald Cut, Marquise, Radiant
  [COLOR]      GIA colour grade            D, E, F, G, H, I, J
  [CLARITY]    GIA clarity grade           FL, IF, VVS1, VVS2, VS1, VS2, SI1, SI2
  [CARAT]      Stone weight in carats      0.25, 0.50, 1.00, 1.50, 2.00
  [DISCOUNT]   Trade discount percentage   0, 5, 10, 15, 20, 25, 30
  ------------ --------------------------- --------------------------------------------------------------------------------

  📝 Developer Note: This prompt is sent to the Claude API, not an image
  AI. It generates a text response that is displayed in the rate result
  UI.

FEATURE 9 — GOLD & SILVER RATE FETCH

This feature fetches live metal rates from an external API (e.g.
metals-api.com or a local data source). No AI image prompt needed. The
following prompt drives the text display layer for rate presentation.

9A — Rate Display Prompt

Prompt:

Today's metal rates for [CITY], [COUNTRY]:

Gold 24K: ₹[GOLD_24K] per gram | ₹[GOLD_24K_TOLA] per tola

Gold 22K: ₹[GOLD_22K] per gram | ₹[GOLD_22K_TOLA] per tola

Gold 18K: ₹[GOLD_18K] per gram | ₹[GOLD_18K_TOLA] per tola

Silver: ₹[SILVER] per gram | ₹[SILVER_TOLA] per tola

Rates updated: [TIMESTAMP]

Note: Rates shown are indicative market rates. Actual buying and selling
rates at jewellers may vary.

Variables:

  ------------- ----------------------------------------------- ---------------------------------------
  Variable      What it represents                              Example values
  [CITY]        User's selected city                            Karachi, Mumbai, Delhi, Dubai, London
  [COUNTRY]     Country of selected city                        Pakistan, India, UAE, UK
  [GOLD_24K]    Live 24K gold rate per gram in local currency   From metals API
  [GOLD_22K]    Calculated as 24K × 0.9167                      From metals API calculation
  [GOLD_18K]    Calculated as 24K × 0.75                        From metals API calculation
  [SILVER]      Live silver rate per gram                       From metals API

FEATURE 11 — IMAGE INPAINTING / TOUCH-UP

The user selects a region of an AI-generated image using a brush tool
and types a correction instruction. The prompt wraps their instruction
in a jewellery-aware context so the AI knows to preserve everything
outside the selected region.

11A — Inpainting Wrapper Prompt

Prompt:

The selected region (masked area) of this jewellery image needs to be
corrected. Apply the following correction only within the masked area:

[USER_CORRECTION]

Preserve everything outside the masked area exactly as it appears — do
not alter any unmasked part of the image. Within the masked area, the
correction must blend seamlessly with the surrounding image in terms of
lighting, colour, texture, and style. The result must be photorealistic
and appear as if the correction was always part of the original image.
High resolution, seamless blending.

Variables:

  ------------------- ---------------------------------------------- -----------------------------------------------------------------------------------------------------------------------------------------------------------------------
  Variable            What it represents                             Example values
  [USER_CORRECTION]   The correction instruction typed by the user   Fix the broken chain link. / Remove the shadow on the left side. / Make the ruby stone brighter. / Add more sparkle to the diamond. / Clean up the background smudge.
  ------------------- ---------------------------------------------- -----------------------------------------------------------------------------------------------------------------------------------------------------------------------

  📝 Developer Note: Pass the original image, the mask image (painted
  region as white on black), and this prompt to the inpainting API
  endpoint. Preserve unmasked pixels exactly.

11B — Quick Fix Presets (Shown as Buttons in UI)

Prompt:

PRESET 1 — Remove Background Smudge:

Remove any smudge, shadow, or dirt marks in the selected area. Replace
with a clean, seamless extension of the surrounding background. No
visible editing marks.

PRESET 2 — Brighten Stone:

Increase the brilliance and light refraction of the gemstone in the
selected area. The stone should appear more vibrant and sparkly while
maintaining its natural colour and cut appearance.

PRESET 3 — Fix Metal Finish:

Smooth out any rough, scratched, or uneven metal surface in the selected
area. The metal should appear polished and consistent with the rest of
the piece.

PRESET 4 — Remove Reflection:

Remove any unwanted reflection, glare, or hot spot in the selected area.
Replace with a natural-looking surface consistent with the surrounding
material.

PRESET 5 — Deepen Shadow:

Add a more natural and defined shadow in the selected area to ground the
jewellery piece on the surface. Shadow should be soft-edged and
directionally consistent with the rest of the image.

  📝 Developer Note: These 5 presets appear as quick-tap buttons in the
  inpainting UI. When the user taps a preset, the preset text is
  automatically injected as [USER_CORRECTION] in the wrapper prompt 11A.
